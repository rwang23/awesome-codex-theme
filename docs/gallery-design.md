# Gallery 设计来源与独立方向

## 为什么重新设计

旧版 Gallery 与 Codex Dream Skin 的展示页 PR #53 共享了过多视觉特征：黑色与纸白底色、荧光强调色、巨大的窄体英文标题、斜体衬线字、编辑式编号和信息条。即使没有逐行复制代码，整体视觉语言仍然过于接近。

PR #53 创建于 2026-07-16，早于本项目的首个提交。这个时间和视觉重合足以说明旧方向不适合作为 Awesome Codex Theme 的长期身份。

参考链接：https://github.com/Fei-Away/Codex-Dream-Skin/pull/53

## 当前视觉系统

新版采用“现代中文主题档案馆”方向：

- 瓷白底、松石绿、朱砂红与少量暖金
- 中文系统无衬线字体为主，不使用巨型窄体英文标题
- Hero 是可切换明暗模式的 Codex 工作区预览窗
- 主题卡片像独立的作品窗口，使用圆角、柔和阴影和明确模式控制
- 系列选择使用横向馆藏卡，而不是编辑编号目录
- 标准、Codex Native 契约与创作 Skill 都进入同一产品叙事
- 默认中文，英文作为完整辅助语言

明确不使用旧方向的荧光酸性色、ticker、斜体衬线强调、巨型全大写标题和编号式海报结构。

这份文档不是为了声明某种风格“归谁所有”，而是给后续贡献者一个可检查的边界，避免 Gallery 再次漂回相近的展示模板。
