# Awesome Codex Theme v0.3.1: safer Full Skin recovery and 15 new themes

> **Beta release / 测试版。** This version improves the Full Skin apply path,
> adds 15 reviewed themes, and refreshes the public evidence catalog. It does
> not claim that every installed ChatGPT or Codex version has already passed a
> real-app Full Skin verification.

## English

### What changed

- **Full Skin now probes the exact installed target on explicit Apply.** A
  detected Stable or Beta build is no longer rejected by a static version-train
  gate. The manager starts only the exact verified executable with loopback-only
  CDP arguments, verifies the process owner and an `app://` target, and applies
  the skin only after the local runtime probe succeeds.
- **Persistence remains fail-closed.** Automatic replay is allowed only for the
  same exact channel and version that already passed that device-local probe. A
  newly updated app stays native until the user explicitly applies again.
- **Failures are clearer.** Recoverable controller failures show a retrying
  state. Terminal launch or CDP errors return their sanitized diagnostic
  immediately instead of being hidden behind the full Apply timeout.
- **15 new themes:** five original xianxia environments, five disclosed,
  non-commercial, unofficial *A Record of a Mortal's Journey to Immortality*
  fan-art scenes, and five original American workspace moods.
- The public catalog is now **68 themes / 136 light-and-dark modes**. Every
  mode has a privacy-safe 1440×810 real capture from the pinned ChatGPT Beta
  test bench `26.715.3651.0`.

### What this fixes—and what it does not claim

This release addresses the misleading behavior where a new Windows machine
could reach a generic “Could not apply the Full Skin” result, including an
opaque POST/CDP access-denied message, after installation. It makes the target
version handling dynamic and exposes a terminal diagnostic promptly.

The reported other-machine error was not available with its full raw diagnostic,
exact selected channel/version, Windows build, or CPU. Therefore this release
does **not** claim that a specific remote `access denied` root cause has been
reproduced or eliminated on every device. If it persists, report the Theme
Manager version, Windows version and CPU, selected Stable/Beta channel and
version, whether the target was already open, and the complete visible
diagnostic. Do not include chats, tokens, or private app data.

### Verification completed before release

- `npm run check` passed: 68 source images, 68 themes, 136 modes,
  68 code-free packages, 136 Full Skin records, 136 Native fallbacks, 136 real
  Beta captures, 20 Node tests, 24 Rust tests, and the Pages build.
- The exact Store package
  `OpenAI.CodexBeta_26.715.3651.0_x64__2p2nqsd0c76g0` was used for the
  new capture batch. The fixture had no selected project, a hidden sidebar, and
  no published private content; the runtime and temporary Beta process were
  removed after capture.
- An isolated browser checked the Gallery in Chinese and English, each new
  collection filter, search, install dialog, and a 390 px mobile viewport with
  no horizontal overflow.

### Important platform note

This Beta uses Tauri updater signatures and an ad-hoc macOS bundle signature.
It does not include Windows Authenticode, Apple Developer ID signing, or Apple
notarization. SmartScreen or Gatekeeper may still show an unknown-publisher
warning. Do not modify ChatGPT application files or disable operating-system
protections to proceed.

Windows has action-after verification on the pinned Beta test bench. macOS
Apple Silicon and Intel artifacts are built and structurally checked in GitHub
Actions, but physical-Mac Full Skin apply, restore, and persistence validation
remain pending.

### 本次更新

- **完整皮肤改为在用户明确点击“应用”后，对准确安装目标做本机探测。** 已检测到的
  Stable 或 Beta 不再受静态版本段白名单限制。管理器只会以回环 CDP 参数启动已核验

- **失败信息更清楚。** 可恢复错误会显示“正在安全重试”；终止性的启动或 CDP 错误会
  立即显示经过净化的诊断，而不是等完整超时结束。
- **新增 15 套主题：** 5 套原创修仙意境、5 套已披露的《凡人修仙传》非官方非商业
  Fan Art，以及 5 套原创美式工作区主题。
- 公共目录现在共有 **68 套主题 / 136 个明暗模式**。每个模式均有固定
  ChatGPT Beta `26.715.3651.0` 测试台生成的隐私安全 1440×810 实机截图。

### 解决了什么，以及不做什么过度承诺

本版本针对另一台 Windows 安装后可能出现“无法应用完整皮肤”、并伴随不透明
POST/CDP access-denied 信息的误导性体验：版本兼容改为动态本机探测，且终止性诊断
会及时显示。

但我们没有拿到该机器完整原始诊断、所选渠道和版本、Windows 版本及 CPU，因此本版本
**不宣称** 已在每台设备上复现或彻底消除某一个具体 `access denied` 根因。若仍然发生，
请反馈 Theme Manager 版本、Windows 版本与 CPU、所选 Stable/Beta 渠道和版本、目标应用
是否原本已开启，以及界面显示的完整诊断；不要附上聊天、Token 或私有应用数据。
