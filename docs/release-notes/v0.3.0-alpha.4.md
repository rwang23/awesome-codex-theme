# Awesome Codex Theme v0.3.0-alpha.4: Clearer themes, dependable artwork

> Beta release / 测试版。本版本重新设计 Theme Manager，移除会遮住背景图的整页色块，并用 106 张新的 ChatGPT Beta 实机截图更新主题目录。

## English

### What is new

- Theme Manager now uses a compact two-column layout. Style tabs, collection filters, a vertically scrolling theme library, the real Beta preview, and the apply controls stay visible in one workspace.
- Full Skin artwork remains on the document background when Codex moves between routes. The main route surface is transparent instead of painting a wide gradient over the image.
- The Codex project sidebar now uses a light or dark tinted material derived from the selected theme's own palette. It no longer falls back to the same plain white panel for every light theme.
- All 53 themes and both modes were recaptured, for 106 real screenshots in total. Catalog revision `2026072101` prevents an installed manager from keeping the previous color-only screenshots.
- The capture harness now recognizes both split and combined model labels. It selected and read back the strongest option actually exposed by this test bench, `5.6 Sol Max`, then restored the user's previous model choice.

### Windows verification

The release candidate was tested against the exact Store package `OpenAI.CodexBeta_26.715.3651.0_x64__2p2nqsd0c76g0`.

- Qinglan Odyssey remained active through Home → Pull requests → Home.
- The 2560×1440 source image remained decoded as a local Blob and attached to the document background.
- The main surface computed to transparent on the tested route.
- The sidebar used the theme's `#DCE9DF` alternate surface instead of generic white.
- The persistent controller remained enabled after the test. No ChatGPT files, `app.asar`, private data, shortcuts, or conversations were modified.

The 106 public captures use a privacy-safe home fixture with no selected project and a hidden sidebar. A separate route test covered the visible project sidebar without publishing its private contents.

### Apply a theme

1. Install Theme Manager. On macOS, move the app to **Applications** before opening it.
2. Pick a style, collection, theme, and light or dark mode. Check the real Beta screenshot.
3. Select the intended ChatGPT target and choose **Apply & Keep Full Skin**.
4. Confirm the one-time persistence notice. The saved selection is reapplied only to the exact supported app and version.
5. Use **Restore Native** to remove the skin and disable the saved selection.

If Apply fails, open a GitHub issue with the Theme Manager version, operating system and CPU, selected ChatGPT channel and version, and the full diagnostic message. Do not edit ChatGPT files or disable system security protections.

### Theme updates and app updates

- Theme updates come from the verified online catalog. New themes and corrected screenshots do not require reinstalling Theme Manager.
- App updates are separate. Runtime, interface, or installer changes use the signed Tauri update channel and require the user's confirmation.

This Beta includes a Tauri updater signature and a macOS ad-hoc bundle signature. It does not yet include Windows Authenticode, Apple Developer ID signing, or Apple notarization, so Windows SmartScreen or macOS Gatekeeper may still show an unknown-publisher warning.

The macOS Apple Silicon and Intel packages are built and structurally checked on GitHub-hosted macOS runners. A physical-Mac Full Skin apply, restore, persistence, Gatekeeper, and notarization test is still pending and is not claimed here.

### 应用主题

1. 安装 Theme Manager。macOS 用户需要先把应用拖入 **Applications**，再从那里打开。
2. 选择风格、系列、主题和明暗模式，并先查看真实 Beta 截图。
3. 选择目标 ChatGPT，点击**应用并保持完整皮肤**。
4. 确认一次性的保持应用提示。保存的主题只会应用到准确匹配且已支持的应用与版本。
5. 想回到原始界面时，点击**恢复原生**；这也会关闭已保存的主题选择。

如果应用失败，请在 GitHub Issue 中附上 Theme Manager 版本、操作系统与 CPU、所选 ChatGPT 渠道和版本，以及完整报错信息。不要修改 ChatGPT 文件，也不要关闭系统安全保护。

### 主题更新与应用更新

- 新主题和修正后的截图通过已验证的在线目录更新，不需要重装 Theme Manager。
- 运行时、界面或安装器发生变化时，才需要更新应用。内置 Tauri 更新通道会先提示，并由用户确认安装。

本测试版包含 Tauri updater 签名和 macOS ad-hoc 包签名，但暂未提供 Windows Authenticode、Apple Developer ID 或 Apple notarization。Windows SmartScreen 或 macOS Gatekeeper 仍可能提示“未知发布者”。

Apple Silicon 与 Intel 安装包会在 GitHub 托管的 macOS Runner 上构建并检查包结构。本版本仍不宣称已经完成真实 Mac 的完整皮肤应用、恢复、持久化、Gatekeeper 与公证测试。
