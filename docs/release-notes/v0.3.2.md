# Awesome Codex Theme v0.3.2

## Release scope

This public beta includes the Full Skin compatibility and diagnostic work from
the v0.3.1 candidate, fifteen new verified themes, and a Gallery metadata fix.

## Delivered

- Explicit **Apply & Keep Full Skin** now performs a local probe for the exact
  detected Stable or Beta target. It is no longer rejected solely because its
  version differs from the pinned evidence build.
- Automatic replay stays conservative: it reuses only the same target/version
  that has already completed a successful local loopback probe.
- Recoverable controller failures show a safe retry state; terminal launch or
  CDP failures return a visible sanitized diagnostic promptly.
- Added fifteen themes: five disclosed non-commercial *A Record of a Mortal's
  Journey to Immortality* fan-art scenes, five original xianxia environments,
  and five original American workspace moods. The Registry now contains 68
  themes and 136 light/dark modes.
- The public Gallery now derives original and Fan Art totals from the Registry,
  and its desktop version label is stamped from the Tauri configuration during
  the static build. This prevents stale release-page metadata after future
  theme additions or version bumps.

## Verification

- `npm run check` passed: 68 source images/themes, 136 modes, 136 Full Skin
  records, 136 Native fallbacks, 136 real Beta captures, 20 Node tests, 24 Rust
  tests, and a static Gallery build with 748 downloadable artifacts.
- `npm run installer:validate` passed for the Windows no-admin helper.
- GitHub Pages built and deployed successfully. An isolated browser confirmed
  the live Gallery shows 68 themes, each of the three new collections filters
  to its five expected themes, and the 390 px viewport has no horizontal
  overflow.

## Compatibility and trust

- Full Skin connects only to a loopback debugging port owned by the selected
  package and an `app://` target. It does not modify application files,
  WindowsApps, user data, or conversations.
- The original remote POST/CDP access-denied report was not directly reproduced
  because its exact target/version and complete visible diagnostic were not
  available. If it persists, attach the complete visible diagnostic, Theme
  Manager version, Windows version/CPU, selected channel/version, and whether
  the target was already open. Do not attach chats, tokens, or private data.
- Windows Authenticode and Apple Developer ID/notarization are not included in
  this beta. Tauri updater signatures protect updater artifacts; macOS bundles
  carry an ad-hoc integrity signature only.

# Awesome Codex Theme v0.3.2（中文）

## 发布范围

这次公开 Beta 包含 v0.3.1 候选版本中的 Full Skin 兼容与诊断改进、十五套新主题，以及
Gallery 元数据修复。

## 本次交付

- 显式点击 **Apply & Keep Full Skin** 时，会对本机检测到的准确 Stable 或 Beta 目标做
  本地探测；不再只因版本不同于固定测试台而拒绝。
- 自动恢复仍保持保守：只有同一目标、同一版本已经完成本地 loopback 探测后才会重放。
- 可恢复控制器故障会显示安全重试状态；终态启动或 CDP 故障会及时显示经过清理的诊断。
- 新增十五套主题：五套已声明、非商业的《凡人修仙传》Fan Art，五套原创修仙环境，
  五套原创美式工作区氛围。Registry 现包含 68 套主题、136 个明暗模式。
- 公共 Gallery 现在从 Registry 实时计算原创与 Fan Art 数量，并在静态构建时从 Tauri
  配置写入桌面版版本号，避免未来加主题或升版本后页面数字漂移。

## 验证

- `npm run check` 通过：68 个源图/主题、136 个模式、136 个 Full Skin 记录、136 个
  Native 回退、136 张真实 Beta 截图、20 个 Node 测试、24 个 Rust 测试，以及包含
  748 个可下载产物的静态 Gallery 构建。
- Windows 免管理员便携助手的 `npm run installer:validate` 通过。
- GitHub Pages 已构建并部署成功。隔离浏览器确认线上 Gallery 显示 68 套主题，三组新
  系列分别筛出预期的五套主题，390 px 视口没有横向溢出。

## 兼容性与信任边界

- Full Skin 只连接由所选安装包拥有的 loopback 调试端口和 `app://` 目标；不会修改
  应用文件、WindowsApps、用户数据或对话内容。
- 原始远端 POST/CDP access-denied 报告没有被直接复现，因为当时缺少准确目标/版本和
  完整可见诊断。如果仍然发生，请提供完整可见诊断、Theme Manager 版本、Windows
  版本/CPU、所选通道/版本，以及目标应用是否已经打开；不要提供聊天、Token 或私有数据。
- 这个 Beta 尚未包含 Windows Authenticode 与 Apple Developer ID/公证。Tauri updater
  签名保护更新产物；macOS 应用包只带 ad-hoc 完整性签名。
